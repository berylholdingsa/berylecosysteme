# Architecture Documentation

## Overview

The Beryl Core API follows clean architecture principles with clear separation of concerns.

## Layers

- **API Layer**: Routes, schemas, middlewares
- **Orchestration Layer**: Business workflow orchestration
- **Adapters Layer**: External API integrations
- **Domain Layer**: Core business logic and entities
- **Shared Layer**: Common utilities, exceptions, enums
- **Observability Layer**: Logging, metrics, tracing

## Strategic Branches

- Fintech: Payments, wallets, transactions
- Mobility: Demand prediction, routing
- ESG: Health data, sustainability scoring
- Social: Recommendations, moderation
